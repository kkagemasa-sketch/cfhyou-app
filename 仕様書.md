## 1. アプリ概要

### 目的
住宅購入を検討する顧客（主にFP・住宅会社営業担当向け）に向けて、長期ライフプランのキャッシュフロー表（CF表）を自動生成するWebアプリケーション。

### 主な機能
- 年次キャッシュフロー表（CF表）の自動生成・表示
- 万が一（ご主人/奥様逝去後）シミュレーション
- 住宅ローン返済計画シミュレーション
- 収支グラフの自動生成
- 複数シナリオの並列管理
- CF表のExcel出力
- JSON形式でのデータ保存・読み込み

### 動作環境
- ビルドツール不要（静的HTML）
- GitHub Pages互換
- スマートフォン対応（iOS対策済み）

### ② 自己資産（その他金融資産）

#### 現預金
| 入力ID | 項目 |
|--------|------|
| `cash-h` | ご主人様 現預金（万円） |
| `cash-w` | 奥様 現預金（万円） |
| `cash-joint` | 共有 現預金（万円） |

**注意**：初期残高 = 現預金合計 - 頭金 - 諸費用（現金払い時）- 引越費用 - 家具費用

#### 財形貯蓄
| 入力ID | 項目 |
|--------|------|
| `zaikei-h-bal` | ご主人様 財形残高（万円） |
| `zaikei-h-monthly` | ご主人様 毎月積立額（万円） |
| `zaikei-h-end` | ご主人様 積立終了年齢 |
| `zaikei-w-bal` | 奥様 財形残高（万円） |
| `zaikei-w-monthly` | 奥様 毎月積立額（万円） |
| `zaikei-w-end` | 奥様 積立終了年齢 |

**計算**：残高は毎年 `bal + monthly × 12` で増加（利息なし）

#### 一時払い保険（動的追加）
`addInsLump(person)` 関数で追加（ご主人/奥様別）。

| 入力ID | 項目 | 説明 |
|--------|------|------|
| `ins-lump-enroll-{p}-{id}` | 加入年齢 | 空欄=現在年齢 |
| `ins-lump-amt-{p}-{id}` | 拠出額（一括）| 万円。**加入年齢時の支出に自動計上** |
| `ins-lump-matage-{p}-{id}` | 満期年齢 | |
| `ins-lump-rate-{p}-{id}` | 想定利率 | %/年。入力時は満期額を自動計算 |
| `ins-lump-matamt-{p}-{id}` | 満期返戻金（固定額） | 万円 |
| `ins-lump-pct-{p}-{id}` | 満期返戻金（拠出額に対する率） | % |

**満期受取計算**（優先順位）：
1. 利率入力あり → `amt × (1 + rate/100)^yrs`
2. 固定額入力あり → そのまま
3. 返戻率入力あり → `amt × pct/100`

**CF表への反映**：
- 支出：`R.insLumpExp` → 加入年齢の年に一括計上
- 収入：`R.insMat` → 満期年齢の年に計上

#### 積み立て保険（動的追加）
`addInsSaving(person)` 関数で追加（ご主人/奥様別）。

| 入力ID | 項目 | 説明 |
|--------|------|------|
| `ins-enroll-{p}-{id}` | 加入年齢 | 空欄=現在年齢 |
| `ins-m-{p}-{id}` | 毎月の保険料 | 万円/月。**支出に自動計上** |
| `ins-age-{p}-{id}` | 満期年齢 | |
| `ins-mat-{p}-{id}` | 満期受取金額 | 万円 |
| `ins-redeem-{p}-{id}` | 解約年齢 | 早期解約時 |
| `ins-redeem-amt-{p}-{id}` | 解約返戻金 | 万円 |

**CF表への反映**：
- 支出：`R.insMonthly` → 加入年齢〜満期年齢（または解約年齢）まで毎年 `monthly × 12`
- 収入：`R.insMat` → 満期（解約あり時は解約年齢）に計上

#### 有価証券（動的追加）
`addSecurity(person)` 関数で追加。

**積立型**：
| 入力ID | 項目 |
|--------|------|
| `sec-bal-{p}-{id}` | 現在残高（万円） |
| `sec-monthly-{p}-{id}` | 毎月積立額（万円/月） |
| `sec-end-{p}-{id}` | 積立終了年齢 |
| `sec-rate-{p}-{id}` | 年利（%）デフォルト5% |
| `sec-redeem-{p}-{id}` | 解約年齢 |
| `sec-nisa-{p}-{id}` | NISA口座かどうか（課税/非課税） |

**一括投資型**：
| 入力ID | 項目 |
|--------|------|
| `sec-stk-bal-{p}-{id}` | 投資額（万円） |
| `sec-stk-age-{p}-{id}` | 投資年齢 |
| `sec-div-{p}-{id}` | 年利（%） |
| `sec-stk-redeem-{p}-{id}` | 売却年齢 |

**課税計算**：課税口座の場合、譲渡益に20.315%を課税（所得税15%+住民税5%+復興特別所得税0.315%）

### 遺族年金計算（自動計算の場合）

#### ご主人様逝去後（奥様への遺族年金）
```
65歳未満の場合：
  遺族厚生年金 = 夫の老齢厚生年金 × 3/4
  + 遺族基礎年金（子が18歳未満いる場合）
  + 中高齢寡婦加算（妻40〜64歳、子なし時 61.43万円）

65歳以降（差額方式・2022年改正後）：
  妻の老齢基礎年金
  + max(夫の老齢厚生年金 × 3/4, 妻の老齢厚生年金)
  + 遺族基礎年金
  + 中高齢寡婦加算
```

**重要**：2022年年金改正後は「差額方式」のみ（2/3・1/2方式は廃止）

#### 遺族基礎年金（2024年度）
```javascript
function calcKiso(n) {
  n=0: 0万円
  n=1: 81.6 + 23.48 = 105.08万円
  n=2: 81.6 + 23.48×2 = 128.56万円
  n≥3: 81.6 + 23.48×2 + 7.83×(n-2)万円
}
// 18歳未満の子がいる場合のみ支給
```

#### 老齢厚生年金の計算（万が一/通常CF共通）
```javascript
const CAREER_FACTOR = 0.75;  // 生涯平均月収 ≒ 現在月収 × 0.75

// 月収入力ありの場合
const hyojun = (capped×12 + bonusCapped) / 12 × CAREER_FACTOR;
const joinM = max((退職年齢 - 加入年齢) × 12, 300);  // 最低300ヶ月
koseiH = hyojun × 5.481/1000 × joinM;

// 月収入力なしの場合（年金設定から逆算）
koseiH = max(0, pSelf - kisoH);
```

**CAREER_FACTOR = 0.75 の根拠**：
現在の月収はキャリアのピーク付近が多く、入社〜退職の生涯平均は概ねピーク時の約75%程度（厚労省・賃金構造基本統計より）。

## 14. 住宅ローン控除計算

### 自動計算モード（`_lctrlDedMode = 'auto'`）
1. `LCTRL_TABLE[year_household][type]` から `[limit, yrs]` を取得
2. 各年のローン残高を計算
3. `cap = min(残高, limit)` で上限キャップ
4. `ctrl = round(cap × 0.007 × 10) / 10`（控除率0.7%）
5. 所得税・住民税から上限計算、`ctrl` とのmin値を採用

### 手動入力モード（`_lctrlDedMode = 'manual'`）
13年分の金額を個別入力可能。

## 15. 年金計算ロジック

### 老齢基礎年金
```javascript
const KISO_FULL = 81.6; // 2024年度満額（万円）
kisoH = round(KISO_FULL × min(退職年齢 - 加入開始年齢, 40) / 40)
kisoW = round(KISO_FULL × min(退職年齢 - 加入開始年齢, 40) / 40)
```

### 老齢厚生年金
```javascript
// 月収入力ありの場合
const CAREER_FACTOR = 0.75;
const capped = min(月収, 65);          // 標準報酬月額の上限
const bonusCapped = min(ボーナス, 300); // 標準賞与額の上限
const hyojun = (capped×12 + bonusCapped) / 12 × CAREER_FACTOR;
const joinM = max((退職年齢 - 加入年齢) × 12, 300); // 最低300ヶ月
koseiH = hyojun × 5.481/1000 × joinM;

// 月収未入力の場合（年金額から逆算）
koseiH = max(0, pension-h - kisoH);
```

### 受給タイミング
- `pS[i]`: ご主人様が `pension-h-receive` 歳以上かつ逝去前のみ計上
- `pW[i]`: 奥様が `pension-w-receive` 歳以上かつ逝去前かつご主人様生存中のみ計上
  - ご主人様逝去後は遺族年金（`survPension`）に統合
